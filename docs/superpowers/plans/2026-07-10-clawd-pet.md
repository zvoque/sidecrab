# Clawd Pet Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** A standalone macOS desktop pet — a hand-coded 8-bit pixel crab that floats always-on-top, is draggable, and animates in reaction to Claude Code activity.

**Architecture:** Tauri v2 app. Rust backend owns the transparent always-on-top window, watches a state file, polls user-idle, and installs/removes its own Claude Code hooks (consent-gated, backed up). A bundled `clawd-pet-hook` Rust binary is what the hooks invoke to write the state file. Frontend is vanilla HTML/CSS/JS: a canvas sprite engine + state machine + behavior/input controllers.

**Tech Stack:** Rust, Tauri v2, `notify` (fs watch), `serde_json`; vanilla JS + `<canvas>`.

## Global Constraints

- macOS only; target current Tauri v2.
- No external runtime at runtime (no node/python) — hook handler is a compiled binary.
- No network calls, no telemetry.
- Original crab art only; product not named "Clawd"; no Clawd wordmark. (See spec §11.)
- Never edit `~/.claude/settings.json` without first-run consent; back it up before first edit; edits additive + idempotent (keyed on our binary path) + removable.
- State file: `<app-config>/clawd-pet/state.json`; sessions: `<app-config>/clawd-pet/sessions.d/`.
- State enum: `idle | thinking | tool | permission | done`.
- Spec: `docs/superpowers/specs/2026-07-10-clawd-pet-design.md` (source of truth).

### Task 7: Wander-when-idle

**Files:**
- Create: `src-tauri/src/idle_monitor.rs`, `src/behavior.js`
- Modify: `src-tauri/src/main.rs`, `src/main.js`

**Interfaces:**
- Produces: Rust polls macOS `HIDIdleTime` (via `ioreg -c IOHIDSystem`), emits `user-idle` / `user-active` on threshold (~60s) crossing. JS `behavior.js` wander controller: on `user-idle` (+ `wanderEnabled`), random-walk the window within the active display bounds playing `run` facing travel dir, with pauses; on `user-active`, scurry back to persisted rest position; Claude-state changes preempt wander; never preempt `permission`.

**Deliverable:** with the toggle on, going idle makes the crab wander; input brings it home; Claude activity overrides.

- [ ] Implement `idle_monitor.rs`: poll every ~2s; parse `HIDIdleTime` nanoseconds→seconds; emit crossing events.
- [ ] Implement `behavior.js`: subscribe to `user-idle`/`user-active`; move window via Tauri position API; pathing within `currentMonitor` bounds; return-home; priority guards.
- [ ] Wire `wanderEnabled` toggle (menu, Task 6) to enable/disable.
- [ ] **Verify:** enable toggle; wait past threshold (or lower it for test) → crab wanders; move mouse → returns home; trigger a Claude turn mid-wander → returns + reacts. Screenshot/screen-recording note.
